Objekt-Orientierte Programmierung (OOP)

Objekt-Orientierte Programmierung in C++ erlaubt die Modellierung von Objekten mit Daten und Funktionen. Diese Lektion behandelt Klassen, Objekte, Konstruktoren und Methoden.

1. Klassen und Objekte

Eine Klasse definiert den Bauplan für Objekte. Ein Objekt ist eine Instanz einer Klasse. 
class Punkt {
public:
    int x, y; // Attribute

    void print() { // Methode
        std::cout << "(" << x << ", " << y << ")" << std::endl;
    }
};

int main() {
    Punkt p1;
    p1.x = 3;
    p1.y = 4;
    p1.print();
    return 0;
}

2. Konstruktoren

Konstruktoren initialisieren Objekte beim Erstellen. 
class Punkt {
public:
    int x, y;
    Punkt(int a, int b) : x(a), y(b) {} // Konstruktor

    void print() {
        std::cout << "(" << x << ", " << y << ")" << std::endl;
    }
};

int main() {
    Punkt p2(5,7);
    p2.print();
    return 0;
}

3. Methoden und Zugriff

Methoden implementieren Verhalten von Objekten. Sichtbarkeit wird über public, private und protected geregelt. 
class Kreis {
private:
    double radius;
public:
    Kreis(double r) : radius(r) {}
    double flaeche() { return 3.14159 * radius * radius; }
};

int main() {
    Kreis k(2.0);
    std::cout << "Fläche: " << k.flaeche() << std::endl;
    return 0;
}

4. Best Practices

  • Attribute sollten private sein, Methoden public
  • Verwende Konstruktoren für Initialisierung
  • Jede Klasse sollte klar definierten Verantwortungsbereich haben

5. Zusammenfassung

  • Klassen definieren Daten & Methoden
  • Objekte sind Instanzen von Klassen
  • Konstruktoren initialisieren Objekte
  • Zugriffsmodifikatoren regeln Sichtbarkeit
